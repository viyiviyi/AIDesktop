# Ai桌面设计方案

一个web页面模拟的类似window11的桌面页面，实际上是一个多agent系统，但以桌面应用程序的方式使用，每个应用程序打开就是一个对话窗口，可以由用户打开来使用也可以由其他APP调用。
分为桌面应用程序和后台服务，后台服务agent没有界面，只能给其他APP或服务调用，APP有界面，被调用时一定会打开界面，并在任务栏显示图标。
每个agent有自己的会话列表，并可以独立管理。

## 后端设计

### API服务

- 使用typescript
- http + sse

#### 功能

- ai模型接口兼容，openai/Anthropic
- 内部使用与模型无关的数据格式，默认定义：文本/音频/图片/视频/文件
- 支持配置多个模型提供商，并对旗下各个模型标记支持的数据格式。
- 支持mcp服务系统，支持动态加载mcp服务并动态注册

##### 内置mcp服务

- 获取agent列表
- 调用agent
- 返回数据给调用的agent并结束

### 数据

#### 数据目录

```
desktop_data/
  apps/
    app1/
      meta.jaon # APP元数据
        - 名称
        - 简介
        - 类型 # 桌面程序/后台服务
        - 图标路径地址
        - 背景图片
        - 使用的模型列表
          - 提供商
          - 模型名
          - 优先级
          - 上下文上限
          - 参数（top_p这种参数）
        - 支持的输入格式
        - 输入说明（提供给其他agent）
        - 内容输出说明（提供给其他agent）
        - 可见的应用程序列表
        - 可见的后台服务列表
        - 可见使用的[mcp/tool]列表
      mcp.json # mcp列表，懒加载，可复用
      skills/ # 模型可使用的技能列表
        include.json 导入了的公共skill，也就是可以将skill安装到公共目录而不是APP目录，减少数据冗余
        allow.json 授权模式、已授权的文件路径、应用列表、服务列表、cli指令列表
        skill1/ # skill符合常规的agent技能要求
        skill2/
        skill...
      data/ # 数据目录
      app.md # agent定义文件
    app2/
    app...
  public_data/
  configs/
    setting.json # 桌面设置，大小、APP布局、主题等设置内容
    mcp.json # 和被
    modes.json
```

#### 数据加载

- 需要自动按照上述数据格式加载桌面图标
- 需要自动加载app.md并注入到APP（agent）上下文。
- 需要根据配置自动加载mcp和skill列表，并注入到agent上下文。

## 前端设计

- 仿mac风格，底部任务栏，顶部状态栏
- 中间是[开始]按钮，点击滑出开始菜单
- 开始菜单左侧显示一列快速启动列表，下方有系统快捷功能（桌面设置/模型配置/mcp管理/技能管理/APP管理）
- 开始菜单右侧是一个对话框，是一个特别的APP
- 长按[开始]按钮也是滑出开始菜单，但进入语音输入模式

### 前端页面开发

- 使用react + antd 单页面应用程序
