## 2. 启动与开发

### 2.1 环境要求
- Node.js 18+

### 2.2 启动命令
```bash
cd client && npm install && npm run dev     # 前端 (端口 5173)
cd server && npm install && npm run dev     # 后端 (端口 3001)
```

前端 Vite 配置了代理 `/api` → `localhost:3001`。

### 2.3 构建
```bash
cd server && npm run build     # esbuild 打包
cd server && npm run sea       # Node.js SEA 独立可执行文件
cd client && npm run build     # 前端构建
```

### 2.4 调试
- 打开"日志"应用可查看实时日志（支持级别/分类过滤）
- 窗口标题栏下方 InjectionBar 显示应用状态摘要
- 后端通过 `serverLogger` 记录日志

## 3. 关键技术决策

### 3.1 纯CSS + CSS变量方案
- 非 CSS-in-JS：Ant Design Token 与 macOS 风格不兼容
- CSS 变量方案性能最优，主题切换无闪烁
- 50+ CSS 变量覆盖所有颜色/间距，双主题（dark/light）一键切换

### 3.2 Pi Agent 引擎
- 自研，非 LangChain
- 核心模式：`streamFn(model, context, options)` 流式调用 LLM
- 每次请求从最新配置读取 bodyParams/headerParams，支持运行时修改
- 通过事件总线（eventBus）驱动流式输出

### 3.3 MCP 两层架构
- **内置 MCP**：`mcpServiceRegistry` 管理，`service.ts` 注册（文件系统、窗口、浏览器等）
- **外部 MCP**：`mcpClientRegistry` 管理，通过 stdio/SSE 连接
- 外部工具名：`mcp_${连接名}_${工具名}`

### 3.4 配置合并
app 的 meta（应用定义）和 config（用户设置）运行时合并，config 优先覆盖 meta。

## 4. 主题系统与配色规范

### 4.1 核心规范

**所有 UI 组件必须通过 CSS 变量引用颜色，禁止硬编码颜色值。**

### 4.2 CSS 变量体系

定义于 `client/src/styles/global.css`，通过 `.theme-dark` / `.theme-light` class 切换。

#### 文本色
```css
--text-primary       /* 主文本 (dark:#fff, light:#1f2937) */
--text-secondary     /* 次要文本 */
--text-tertiary      /* 第三级文本 */
```

#### 背景色
```css
--desktop-bg, --window-bg, --window-title-bg, --menu-bar-bg, --dock-bg
--bg-primary, --bg-secondary, --bg-tertiary, --msg-bg, --header-bg
```

#### 表单
```css
--input-bg, --input-border, --input-text, --input-placeholder
--select-bg, --select-text
```

#### 边框
```css
--border-primary, --border-secondary
```

#### 状态色
```css
--accent-color (#0078d4), --success-*, --error-*, --warning-*
```

### 4.3 ✅ 正确做法
```css
.my-component {
  color: var(--text-primary);
  background: var(--window-bg);
  border: 1px solid var(--border-primary);
}
```

### 4.4 ❌ 禁止做法
```css
.my-component {
  color: #ffffff;               /* 亮色主题下应是深色 */
  background: #1e1e2e;          /* 不跟随主题 */
}
```

### 4.5 内联样式规范

```typescript
// ✅ 正确
<div style={{ color: 'var(--text-primary)', background: 'var(--bg-secondary)' }}>
// ❌ 错误
<div style={{ color: '#ffffff', background: 'rgba(255,255,255,0.05)' }}>
```

### 4.6 新组件主题兼容检查清单

- [ ] 文字颜色使用 `var(--text-primary/secondary/tertiary)`
- [ ] 背景使用 `var(--bg-*)` 或 `var(--window-bg)`
- [ ] 边框使用 `var(--border-primary/secondary)`
- [ ] 输入框使用 `var(--input-*)`
- [ ] 状态提示使用 `var(--success-*)` / `var(--error-*)` / `var(--warning-*)`
- [ ] 无 `#xxx`、`rgb()`、`rgba()` 硬编码
- [ ] 弹窗背景使用 `var(--window-bg)`，覆盖层使用 `rgba(0,0,0,0.4)`

## 5. 前端开发规范

### 5.1 窗口布局模式

```css
.xxx-window { flex: 1; display: flex; flex-direction: column; overflow: hidden; min-height: 0; }
.xxx-body   { flex: 1; overflow-y: auto; min-height: 0; }
.xxx-header { /* 固定高度 */ }
.xxx-footer { /* 固定高度 */ }
```

**中间内容区域必须用 `flex: 1; overflow-y: auto; min-height: 0`，确保滚动条只在内容区出现。**

### 5.2 窗口高度自适应

- ❌ 禁止在列表/内容区使用 `max-height` 固定值（如 `max-height: 200px`）
- ❌ 禁止使用 `height: 100%` 替代 `flex: 1`
- ✅ 使用 `flex: 1` + `min-height: 0` 确保 flex 子项正确收缩

### 5.3 InjectionBar 规范

- 位于标题栏和消息列表之间
- 每个 block 的展开详情使用 Markdown 格式
- 颜色由 CSS class 的 `--inj-color` 变量控制，无硬编码
- 刷新通过变更组件的 `key` 触发重新挂载

### 5.4 流式消息渲染

- 使用 `MarkdownView` 组件渲染（支持 GFM、代码高亮、链接新窗口）
- 流式消息也通过 `MarkdownView` 组件渲染

### 5.5 应用设置"记忆"标签页

- 在 `AppSettingsWindow` 中标签页按钮加在"提示"后面
- `activeTab` 类型包含 `'memory'`
- body 中渲染 `<MemoryPanel appId={appId} scope="app" />`

## 6. 后端开发规范

### 6.1 日志规范

使用 `serverUtils.logger` 记录日志：

```typescript
serverLogger.info('agent', `工具调用: ${toolName}`, { toolCallId, args });
serverLogger.ai(source, '>>> 请求', { messages: count });
serverLogger.ai(source, '<<< 响应', { stopReason, toolCalls });
serverLogger.api(method, url, status, duration?);
```

**工具调用和 AI 请求/响应是强制记录点。**

### 6.2 事件驱动

| 事件 | 触发 | 说明 |
|------|------|------|
| `text_chunk` | 流式文本块 | 前端追加到 streamingText |
| `tool_call` | 工具执行开始 | 前端显示 toolCall 卡片 |
| `tool_result` | 工具执行结束 | 前端显示 toolResult |
| `message_end` | 完整消息结束 | 前端保存到 messages |

## 7. 外部 MCP 工具命名规范

### 7.1 Key 格式

```typescript
// 配置存储
const appToolKey = `external:${safeConnName}:${tool.name}`;
// 例: external:Browser:browser_navigate

// AI Agent 工具名
const agentToolName = `mcp_${safeConnName}_${safeToolName}`;
// 例: mcp_Browser_browser_navigate
```

`safeConnName` = `connName.replace(/[^a-zA-Z0-9_-]/g, '_')`
`connName` = `client.serverInfo?.name || connectionId`

### 7.2 禁止行为

**禁止在工具 key 中使用 UUID 作为连接标识符。** 使用连接名替换。

### 7.3 前端渲染

```typescript
// AppSettingsWindow.tsx — 外部 MCP 工具 checkbox key
const safeConnName = connName.replace(/[^a-zA-Z0-9_-]/g, '_');
const toolKey = `external:${safeConnName}:${tool.name}`;
```

## 9. 常见问题

### 9.1 记忆不生效

1. 检查应用设置 → 工具标签页中是否已勾选 `mcp.memory`
2. 检查应用设置 → 记忆标签页中是否有数据
3. 检查日志中是否有 `[memory]` 标签的条目
4. 重新发送消息触发 system prompt 重建

### 9.2 开始菜单上次对话不还原

- 关闭菜单时组件被 unmount，重新打开通过 `loadConversations()` 重新加载
- `sessionStorage.getItem('startmenu_last_conv')` 记录上次会话 ID
- 检查后端 `/api/apps/{appId}/conversations` 接口是否正常

### 9.3 窗口内容双滚动条

检查 CSS 链路，确保中间所有 flex container 有 `min-height: 0`：
```css
.window-content { flex: 1; overflow: hidden; min-height: 0; }
.xxx-window { flex: 1; min-height: 0; }
.xxx-body { flex: 1; overflow-y: auto; min-height: 0; }
```

### 9.4 工具名称带 UUID

- 旧格式 `external:UUID:toolName` 在新代码下自动失效
- 需在应用设置 → 工具标签页重新勾选对应 MCP 工具并保存
- 新格式：`external:连接名:工具名`
