# AIDesktop 2.0 开发文档 — 记忆与目标管理系统

> 本文档定义 AIDesktop 2.0 版本的记忆系统架构、数据模型、目标树结构、标签体系、加载机制和注入可视化方案。开发者应在实现 2.0 功能前完整阅读本文档。

---

## 1. 设计目标

### 1.1 核心需求

- Agent 能在会话间记住用户偏好、事实和上下文
- 长期记忆与当前会话上下文分离，防止 token 膨胀
- 会话级拥有特殊的目标树结构，驱动 AI 的任务管理
- 支持按标签分类、按层级加载、按条件查找
- 前端显示注入标记，让用户清楚 AI 当前拥有哪些上下文
- 纯文件系统存储，不引入外部数据库/向量引擎

### 1.2 应用场景

| 场景 | 描述 | 对应记忆级别 |
|------|------|-------------|
| 用户说"叫我小王" | Agent 记住用户名 | 应用级 fact |
| 用户说"我一般用中文提问" | Agent 记住语言偏好 | 应用级 preference |
| 用户说"这个项目用 React" | Agent 记住项目技术栈 | 应用级 context |
| 用户说"我们的目标是把页面做漂亮" | Agent 设置一级目标 | 会话级 goal |
| 用户说"先完成登录页面" | Agent 设置二级目标 | 会话级 goal |
| 用户说"当前在做表单验证" | Agent 更新三级目标 | 会话级 goal |
| 用户说"登录页面做好了" | Agent 归档二级目标 | 会话级 goal → archived |
| Agent 正在处理一个多步骤任务 | Agent 记录中间状态 | 会话级 context |

## 2. 数据模型

### 2.1 记忆条目 (MemoryEntry)

```typescript
interface MemoryEntry {
  // === 标识 ===
  id: string;                    // UUID
  type: 'fact' | 'preference' | 'context' | 'goal';  // 记忆类型

  // === 多级键名 ===
  // 键名采用 dot notation 多级结构：
  // user.name            — 用户的名称
  // user.language        — 用户的语言偏好
  // project.tech         — 项目的技术栈
  // project.rule.naming  — 项目的命名规范
  // goal/level1          — 一级目标（会话级专用）
  // goal/level2          — 二级目标（会话级专用）
  // goal/level3          — 三级目标（会话级专用）
  key: string;

  // === 值 ===
  value: string;                 // 结构化值
  content: string;               // 自然语言描述（用于全文检索）

  // === 标签体系 ===
  // 标签也支持层级：category/subcategory
  // 例如：["user/info", "importance/high", "goal/active"]
  tags: string[];

  // === 元数据 ===
  createdAt: string;             // ISO 时间戳
  updatedAt: string;
  source: 'user' | 'agent' | 'system';  // 记忆来源
  scope: 'app' | 'conversation';         // 记忆范围
  conversationId?: string;       // 会话级记忆所属会话 ID
  ttl?: number;                  // 可选过期时间（秒），到期后自动清理
}
```

### 2.2 目标条目 (GoalEntry) — 会话级专用

目标条目与普通记忆共存在同一个 `memories.json` 中，通过 `type: 'goal'` 和 `tags` 中的 `goal/active` / `goal/archived` 区分。

## 3. 核心架构

### 3.1 记忆层级结构

```
应用级 (app scope)
├── user/                 # 用户相关信息
│   ├── name              # 用户名
│   └── language          # 用户语言偏好
├── project/              # 项目相关信息
│   ├── tech              # 技术栈
│   └── rules             # 项目规范
└── preferences/          # 用户偏好
    ├── theme             # 主题偏好
    └── format            # 输出格式偏好

会话级 (conversation scope)
├── goal/                 # 目标树（type='goal'）
│   ├── level1            # 一级目标（唯一活跃）
│   ├── level2            # 二级目标（唯一活跃）
│   └── level3            # 三级目标——当前待办（唯一活跃）
│   // 已归档的 goal 也在此，tags=["goal/archived"]
├── temp/                 # 临时上下文
│   ├── current_task      # 当前任务描述
│   └── last_reference    # 上次引用
└── working/              # 工作中间状态
    ├── files_referenced  # 本次会话引用的文件
    └── pending_decisions # 待定决策
```

### 3.2 目标树结构与规则

```
┌─────────────────────────────────────┐
│      一级目标（goal/level1）         │
│  唯一活跃，是整个会话的大方向         │
│  例如："打造一个美观易用的桌面应用"    │
└────────────────┬────────────────────┘
                 │ 当 level1 更新时，level2 和 level3 的旧值归档
                 ▼
┌─────────────────────────────────────┐
│      二级目标（goal/level2）         │
│  唯一活跃，是当前阶段的具体方向        │
│  例如："完成用户登录页面"             │
└────────────────┬────────────────────┘
                 │ 当 level2 更新时，level3 的旧值归档
                 ▼
┌─────────────────────────────────────┐
│      三级目标（goal/level3）         │
│  唯一活跃，是当前的待办事项           │
│  例如："实现表单验证逻辑"             │
└─────────────────────────────────────┘
```

**变更规则：**

| 操作 | 效果 |
|------|------|
| 设置/更新 level1 | 旧 level1 归档，level2 和 level3 **全部归档** |
| 设置/更新 level2 | 旧 level2 归档，level3 归档 |
| 设置/更新 level3 | 旧 level3 归档 |
| 完成 level3 | level3 归档，清空（等待新 level3） |
| 完成 level2 | level2 归档，level3 归档 |
| 完成 level1 | level1 归档，level2 和 level3 归档 |

**归档规则：**
- 归档不删除，仅将 tags 中的 `goal/active` 改为 `goal/archived`
- 已归档的目标不在 system prompt 中注入
- 可通过 MCP 工具 `recallArchivedGoals` 单独查询

### 3.3 标签体系

| 标签 | 含义 | 示例 |
|------|------|------|
| `importance/low` | 低重要性 | 临时偏好 |
| `importance/normal` | 普通重要性 | 一般事实 |
| `importance/high` | 高重要性 | 关键约束、目标 |
| `user/info` | 用户个人信息 | 名字、年龄 |
| `user/preference` | 用户偏好 | 主题、语言 |
| `project/tech` | 项目技术栈 | 框架、语言 |
| `project/rule` | 项目规范 | 命名规范 |
| `project/constraint` | 项目约束 | 必须遵守的规则 |
| `temporary` | 临时记忆 | 会话中间状态 |
| `goal/active` | 活跃目标 | 当前目标 |
| `goal/archived` | 已归档目标 | 已完成/被替代的目标 |

### 3.4 标签自动添加规则

- Agent 每次 `remember` 时自动添加 `source/agent` 标签
- 用户通过 mcp.memory 保存的添加 `source/user` 标签
- 重要性高的自动加 `importance/high` 标签
- 会话级记忆自动加 `scope/conversation` 标签
- 目标条目自动加 `goal/` 类标签

## 4. API 设计

### 4.1 MemoryService（后端服务）

```typescript
class MemoryService {
  // === 通用 CRUD ===

  /** 添加一条记忆 */
  async remember(scope: MemoryScope, appId: string, entry: {
    type: MemoryType;
    key: string;              // 支持 dot notation
    value: string;
    content?: string;
    tags?: string[];
    source?: 'user' | 'agent' | 'system';
    ttl?: number;
  }, convId?: string): Promise<MemoryEntry>;

  /** 查询记忆（支持多级过滤） */
  async recall(scope: MemoryScope, appId: string, query: {
    key?: string;
    keyPrefix?: string;       // 键前缀，如 "user." 匹配所有 user 相关
    type?: MemoryType;
    tags?: string[];          // 标签 AND 过滤
    tagsAny?: string[];       // 标签 OR 过滤
    search?: string;          // 全文检索
    limit?: number;
    offset?: number;
  }, convId?: string): Promise<MemoryEntry[]>;

  /** 按层级加载（加载指定层级及其下级） */
  async recallByPrefix(scope: MemoryScope, appId: string, keyPrefix: string, convId?: string): Promise<MemoryEntry[]>;

  /** 更新记忆 */
  async update(scope: MemoryScope, appId: string, entryId: string, updates: Partial<Pick<MemoryEntry, 'value' | 'content' | 'tags' | 'key' | 'type'>>, convId?: string): Promise<MemoryEntry | null>;

  /** 删除一条记忆 */
  async forget(scope: MemoryScope, appId: string, entryId: string, convId?: string): Promise<boolean>;

  /** 按标签批量删除 */
  async forgetByTag(scope: MemoryScope, appId: string, tag: string, convId?: string): Promise<number>;

  /** 列出所有标签 */
  async listTags(scope: MemoryScope, appId: string, convId?: string): Promise<string[]>;

  /** 获取统计 */
  async stats(scope: MemoryScope, appId: string, convId?: string): Promise<{
    total: number;
    byType: Record<string, number>;
    byTag: Record<string, number>;
  }>;

  /** 获取所有记忆 */
  async getAll(scope: MemoryScope, appId: string, convId?: string): Promise<MemoryEntry[]>;

  // === 目标树专用方法 ===

  /** 获取当前活跃的目标树 */
  async getActiveGoals(appId: string, convId: string): Promise<{
    level1: MemoryEntry | null;
    level2: MemoryEntry | null;
    level3: MemoryEntry | null;
  }>;

  /** 设置一级目标（旧 level1 归档，level2/level3 全部归档） */
  async setLevel1Goal(appId: string, convId: string, value: string, source: string): Promise<MemoryEntry>;

  /** 设置二级目标（旧 level2 归档，level3 归档） */
  async setLevel2Goal(appId: string, convId: string, value: string, source: string): Promise<MemoryEntry>;

  /** 设置三级目标/当前待办（旧 level3 归档） */
  async setLevel3Goal(appId: string, convId: string, value: string, source: string): Promise<MemoryEntry>;

  /** 完成当前活跃的目标 */
  async completeGoal(appId: string, convId: string, level: 1 | 2 | 3): Promise<void>;

  /** 查询已归档的目标 */
  async getArchivedGoals(appId: string, convId: string): Promise<MemoryEntry[]>;

  // === 注入 ===

  /** 构建用于 system prompt 注入的记忆块 */
  async buildMemoryBlock(appId: string, options?: {
    convId?: string;
    keyPrefix?: string;
    maxEntries?: number;
    minImportance?: 'low' | 'normal' | 'high';
  }): Promise<string>;
}
```

## 5. System Prompt 注入策略

### 5.1 注入位置

在 `buildSystemPrompt` 函数中，技能注入之后、返回之前注入记忆块。
目标树始终注入在记忆块最顶部，使用独立的格式化方法。

### 5.2 分级加载算法

```typescript
async buildMemoryBlock(appId: string, options?: {
  convId?: string;
  keyPrefix?: string;
  maxEntries?: number;
  minImportance?: 'low' | 'normal' | 'high';
}): Promise<string> {
  const maxEntries = options?.maxEntries ?? 30;
  let blocks: string[] = [];

  // 1. 始终注入活跃目标树
  if (options?.convId) {
    const goals = await this.getActiveGoals(appId, options.convId);
    if (goals.level1 || goals.level2 || goals.level3) {
      let goalBlock = '\n## 当前目标\n';
      if (goals.level1) {
        goalBlock += `【一级目标】${goals.level1.value}\n`;
        if (goals.level2) {
          goalBlock += `  【二级目标】${goals.level2.value}\n`;
          if (goals.level3) {
            goalBlock += `    【三级目标】${goals.level3.value} ← 当前待办\n`;
          }
        }
      }
      blocks.push(goalBlock);
    }
  }

  // 2. 加载应用级记忆（按重要性排序）
  const allApp = await this.getAll('app', appId);
  const sorted = allApp
    .filter(e => {
      if (options?.keyPrefix && !e.key.startsWith(options.keyPrefix)) return false;
      if (options?.minImportance) {
        const levels = ['high', 'normal', 'low'];
        const minIdx = levels.indexOf(options.minImportance);
        const entryIdx = this.getImportanceLevel(e);
        if (entryIdx > minIdx) return false;
      }
      return true;
    })
    .sort((a, b) => this.getImportanceLevel(a) - this.getImportanceLevel(b)
      || b.updatedAt.localeCompare(a.updatedAt));

  const selected = sorted.slice(0, maxEntries);
  if (selected.length > 0) {
    blocks.push(formatMemoryBlock(selected));
  }

  // 3. 加载会话级记忆（不含 goal）
  if (options?.convId) {
    const allConv = await this.getAll('conversation', appId, options.convId);
    const convNonGoal = allConv
      .filter(e => e.type !== 'goal')
      .slice(0, 10);
    if (convNonGoal.length > 0) {
      blocks.push('\n## 会话上下文\n');
      for (const m of convNonGoal) {
        blocks.push(`- ${m.key}: ${m.value}${m.content ? ` — ${m.content}` : ''}\n`);
      }
    }
  }

  return blocks.join('\n') + '\n';
}
```

**分级加载策略：**

| 条件 | 注入内容 |
|------|----------|
| 首次对话 | 全部 high + normal 重要性 |
| 已有历史 | 仅 high + 最近更新的 normal |
| 用户明确提及标签/前缀 | 按需加载指定前缀下所有 |
| 目标树 | **始终注入**，显示在记忆块最顶部 |

## 6. 注入可视化（前端）

### 6.1 注入标记系统

所有注入到 system prompt 的内容需要在**对话界面的输入框上方**显示标记，标明当前 AI 拥有哪些上下文信息。

设计已在后端 `pi-agent-session.ts` 中预实现了 `buildInjectionSummary` 函数和 `InjectionBlock` 接口：

```typescript
// pi-agent-session.ts — 已存在
export interface InjectionBlock {
  source: 'app' | 'agents' | 'skills' | 'memory' | 'goal' | 'prompt';
  label: string;       // 显示标签，如"记忆"、"技能"、"目标"
  title: string;       // 标题摘要
  detail: string;      // 详细内容（供展开查看）
}

export async function buildInjectionSummary(app: App, convId?: string): Promise<InjectionBlock[]>;
```

来源覆盖情况：

| source | label | 说明 | 状态 |
|--------|-------|------|------|
| `app` | 应用定义 | app.md 的内容 | ✅ 已实现 |
| `agents` | 可调用的 Agent | visibleApps 列表 | ✅ 已实现 |
| `skills` | 已加载的技能 | 用户勾选的技能 | ✅ 已实现 |
| `memory` | 记忆 | 应用级 + 会话级记忆 | ✅ 已实现 |
| `goal` | 当前目标 | 会话目标树（活跃） | ⏳ 待实现 |
| `prompt` | 系统提示词 | 当前 model + provider | ⏳ 待实现 |

### 6.2 InjectionBlock — goal 来源实现

在 `buildInjectionSummary` 中添加 goal 块：

```typescript
// 5. 当前目标
if (convId) {
  try {
    const { memoryService } = await import('../services/memory.js');
    const goals = await memoryService.getActiveGoals(app.meta.id, convId);
    if (goals.level1 || goals.level2 || goals.level3) {
      let detail = '';
      if (goals.level1) detail += `【一级】${goals.level1.value}\n`;
      if (goals.level2) detail += `【二级】${goals.level2.value}\n`;
      if (goals.level3) detail += `【三级】${goals.level3.value}`;
      blocks.push({
        source: 'goal',
        label: '当前目标',
        title: goals.level3 ? '有活跃待办' : '有活跃目标',
        detail: detail.trim(),
      });
    }
  } catch {}
}
```

**组件（`client/src/components/InjectionBar.tsx`）：**

```tsx
import { useState, useEffect } from 'react';
import * as api from '../services/api';

const SOURCE_CONFIG: Record<string, { icon: string; color: string }> = {
  app:     { icon: '📋', color: 'var(--blue)' },
  agents:  { icon: '🤖', color: 'var(--purple)' },
  skills:  { icon: '📚', color: 'var(--green)' },
  goal:    { icon: '🎯', color: 'var(--orange)' },
  memory:  { icon: '🧠', color: 'var(--cyan)' },
  prompt:  { icon: '⚙️', color: 'var(--gray)' },
};

export function InjectionBar({ appId, convId }: { appId: string; convId: string | null }) {
  const [blocks, setBlocks] = useState<api.InjectionBlock[]>([]);
  const [expanded, setExpanded] = useState<string | null>(null);

  const load = async () => {
    if (!appId || !convId) return;
    try {
      const data = await api.getInjections(appId, convId);
      setBlocks(data.blocks || []);
    } catch {}
  };

  useEffect(() => { load(); }, [appId, convId]);

  // 可通过事件总线监听对话完成后刷新

  return (
    <div className="injection-bar">
      {blocks.map(b => {
        const cfg = SOURCE_CONFIG[b.source] || { icon: '📎', color: 'var(--gray)' };
        return (
          <div
            key={b.source}
            className="injection-tag"
            style={{ borderColor: cfg.color }}
            onClick={() => setExpanded(expanded === b.source ? null : b.source)}
          >
            <span>{cfg.icon}</span>
            <span>{b.label}</span>
            <span className="injection-title">{b.title}</span>
          </div>
        );
      })}
      {expanded && (
        <div className="injection-detail">
          <pre>{blocks.find(b => b.source === expanded)?.detail}</pre>
        </div>
      )}
    </div>
  );
}
```

在 `Window.tsx` 和 `StartMenu.tsx` 的输入框上方插入 `<InjectionBar>`。

## 7. 边界情况与约束

### 7.1 记忆数量控制

| 项目 | 上限 | 超出策略 |
|------|------|----------|
| 应用级记忆 | 500 条 | 裁剪最旧的 normal/low |
| 会话级记忆（非 goal） | 100 条 | 裁剪最旧的 |
| 目标树活跃条目 | 3 条（level1+2+3） | 唯一性保证 |
| 已归档目标 | 无上限 | — |
| System prompt 注入 | 30 条应用级 + 10 条会话级 + 目标树 | 按重要性筛选 |
| 单条 value/content | 1000 字符 | 超长截断 |

### 7.2 标签规范

- 格式：`category/subcategory`（小写英文，连字符分隔）
- 系统保留前缀：`importance/`、`source/`、`scope/`、`goal/`
- 用户自定义标签建议用应用名做前缀：`myapp/tagname`

### 7.3 并发安全

- 每次 save 增加 `version` 字段
- 写操作时比较 version，不匹配则重新加载
- 单用户场景，不做复杂锁机制

### 7.4 过期清理

- 每次加载记忆时检查 `ttl`
- 过期的记忆在加载时自动删除

### 7.5 与其他系统的关系

| 系统 | 关系 |
|------|------|
| **Skill 系统** | 技能是静态知识，记忆是动态上下文。技能注入在 prompt 头部，记忆在尾部 |
| **Conversation 系统** | 会话记录原始对话，记忆是提炼的精华摘要 |
| **Workspace 系统** | 工作目录是文件级上下文，记忆是语义级上下文 |
