# 工作目录与会话工作工具设计

## 概述

将内置 MCP 服务分为两类：
- **内置工具**（`mcp.*`）：用于维护系统本身（`mcp.agent`, `mcp.window`, `mcp.settings`, `mcp.sleep`, `mcp.exec`, `mcp.http`, `mcp.browser`）
- **工作工具**（`workspace.*`）：用于用户的工作任务，需要设置工作目录后才能使用（`workspace.code`, `workspace.form`）

## 核心概念

### 工作目录

每个会话可以有独立的工作目录。工作目录是用户批准的绝对路径。

- 第一次使用工作工具时，弹出表单让用户选择并确认工作目录
- 设置后，工作工具的所有相对路径都相对于工作目录
- 工作目录跟随 callChain：A 调用 B 时，B 的会话继承 A 的工作目录
- 会话标题旁边显示工作目录

### 工具分类

**内置工具**（`mcp.*`）：
- 系统维护用途，无需用户授权
- 不限制路径（已有实现）

**工作工具**（`workspace.*`）：
- 用户工作任务用途
- 必须在会话设置了工作目录后才能使用
- 路径限制在工作目录内

## 数据结构

### Conversation 增加字段

```typescript
interface Conversation {
  // ...现有字段
  workspaceDir?: string | null;  // 会话的工作目录绝对路径
}
```

### MCPService 增加分类字段

```typescript
interface MCPService {
  name: string;
  description: string;
  methods: string[];
  category?: 'builtin' | 'workspace';  // 默认 'builtin'
}
```

## 实现步骤

### 1. 类型定义
- types/index.ts: Conversation 增加 workspaceDir
- types/index.ts: MCPService 增加 category

### 2. mcp/service.ts
- 将 `mcp.code` 重命名为 `workspace.code`，`mcp.form` 重命名为 `workspace.form`
- 新建 `workspace` 分类的服务注册表
- 新增 `mcp.workspace` 服务（`setDir`, `getDir` 方法）
- 工作工具的方法中，从会话获取 workspaceDir 作为根路径

### 3. pi-tools.ts
- `buildPiToolsForApp` 只暴露内置工具
- 工作工具由单独的构建函数创建，需要会话上下文

### 4. pi-agent-session.ts
- `runAgentAsync` 中传递会话的工作目录到工具执行上下文
- 调用链传递工作目录

### 5. Conversation 前端
- `workspace.requestAccess` 服务：AI 调用时向用户弹出目录选择表单
- `workspace.setDir`：设置工作目录
- 会话标题显示工作目录
